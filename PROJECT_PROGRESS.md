# تقرير تقدم مشروع نظام إدارة مهام متاجر سلة (Salla Task Management)

هذا المستند يلخص الحالة الراهنة للمشروع، التقنيات المستخدمة، والمراحل المنجزة حتى تاريخ اليوم.

## 🚀 التقنيات المستخدمة (Tech Stack)

تم بناء النظام باستخدام أحدث التقنيات لضمان السرعة، القابلية للتوسع، والأمان:

*   **الواجهة الأمامية (Frontend):** React 19 مع Vite و TypeScript لضمان أداء عالٍ وأمان الأنواع.
*   **التصميم (Styling):** Tailwind CSS 4 مع Vanilla CSS لتصميم عصري ومتجاوب (RTL) — Mobile-First.
*   **الخادم الخلفي (Backend):** Node.js مع Express و TypeScript.
*   **قاعدة البيانات (Database):** SQLite (تطوير) مع استخدام Prisma كـ ORM — جاهز للترقية لـ PostgreSQL.
*   **التخزين:** Supabase Storage + Local Uploads للوثائق والصور.
*   **الذكاء الاصطناعي (AI):** تكامل مع Gemini AI (موديلات 2.5 Flash + Flash-Lite) لتوليد الهويات التجارية والشعارات.
*   **الأمان:** نظام JWT للمصادقة + bcrypt لتشفير كلمات المرور + RBAC لتقييد الوصول.
*   **التحقق من المدخلات:** Regex-based validation مزدوج (Frontend + Backend).
*   **إدارة الحالة:** Zustand مع localStorage persistence.

## 📊 ملخص هيكلة قاعدة البيانات (Database Schema)

تتكون قاعدة البيانات من النماذج الرئيسية التالية:

1.  **User:** يدير بيانات المستخدمين (عملاء، مدراء حسابات، مختصي SEO، مصممين، مبرمجين، أدمن).
2.  **ClientInfo:** يخزن البيانات القانونية (الهوية، الآيبان، الجوال) والبيانات التجارية — مع حقول `docsApproved` و `needsLegalExtraction` لسير عمل الموافقة.
3.  **AIProposal:** يحتوي على مقترحات الأسماء، لوحات الألوان، ونبرة الصوت والشعارات المولدة بواسطة AI.
4.  **Ticket:** يمثل الطلب الرئيسي ويربط العميل بالمراحل المختلفة — مع حقل `isArchived` للأرشفة.
5.  **StoreDetails:** يخزن تفاصيل المتجر التقنية بعد التأسيس (رابط المتجر، الدومين).
6.  **Notification:** نظام الإشعارات المرتبط بالمستخدمين والتذاكر.
7.  **AuditLog:** سجل التدقيق لتتبع جميع الإجراءات في النظام.

## 🏆 المميزات المنجزة (Completed Features)

### المرحلة 1 — الأساسيات (Phase 1)
*   ✅ **بوابة العميل المتطورة:** نموذج إدخال بيانات مكون من خطوتين (بيانات قانونية ➔ تفاصيل المتجر).
*   ✅ **بوابة التحقق القانوني:** منطق شرطي (مسار A: عنده وثيقة / مسار B: يحتاج استخراج) مع توجيه للدعم عبر واتساب.
*   ✅ **محرك توليد الهوية (Gemini AI):** نظام إنتاجي يولد أسماء وألوان ابتكارية بناءً على مجال العمل.
*   ✅ **توليد الشعارات (SVG):** شعارات ديناميكية عبر AI مع دعم التوليد المتكرر وتتبع النسخ.
*   ✅ **تعديل الهوية التفاعلي:** إمكانية تعديل الأسماء المقترحة أو اختيار ألوان مخصصة قبل الاعتماد.
*   ✅ **لوحة تحكم العميل (Dashboard):** تتبع لحظي لحالة الطلب عبر Stepper مرئي (6 خطوات).

### المرحلة 2 — المصادقة والأمان (Phase 2)
*   ✅ **نظام مصادقة متطور (Two-Step Login):** الخطوة 1: التحقق من البريد → الخطوة 2: كلمة المرور (للموظفين فقط).
*   ✅ **تغيير كلمة المرور:** صفحة إعدادات متاحة لجميع الموظفين مع تحقق bcrypt.
*   ✅ **تشفير كلمات المرور:** bcrypt hashing لكل كلمات مرور الموظفين.
*   ✅ **التحقق من المدخلات (Regex Validation):** طبقة مزدوجة (Backend + Frontend) لكل الحقول الحساسة.
*   ✅ **تقييد الإدخال في الوقت الفعلي:** أرقام فقط للهوية/الجوال، uppercase تلقائي للآيبان، maxLength لكل حقل.

### المرحلة 3 — RBAC والأمان المتقدم (Phase 3)
*   ✅ **RBAC (Role-Based Access Control):** 6 أدوار (ADMIN, ACCOUNT_MANAGER, SEO, DESIGNER, DEVELOPER, CUSTOMER).
*   ✅ **StaffGuard:** يمنع العملاء من الوصول لـ `/dashboard`.
*   ✅ **ClientGuard:** يوجّه المستخدمين غير المكتملين للـ onboarding.
*   ✅ **حماية الأزرار الحساسة:** أزرار الحذف والأرشفة تظهر فقط لـ ADMIN و ACCOUNT_MANAGER.

### المرحلة 4 — Webhook وسير العمل القانوني (Phase 4)
*   ✅ **Adtopia Webhook:** استقبال بيانات العملاء عبر `POST /api/webhooks/adtopia` مع Bearer token authentication.
*   ✅ **تعبئة مسبقة للبيانات:** الاسم، البريد، والجوال يُعبّأ تلقائياً من بيانات الـ webhook ويكون للقراءة فقط.
*   ✅ **سير عمل الموافقة على الوثائق:** زر "اعتماد الوثائق" في لوحة الأدمن → يظهر "تمت المراجعة" عند العميل.
*   ✅ **Smart Detection:** كشف ذكي لحالات الاستخراج حتى لو الحقل لم يُحفظ بشكل صحيح.

### المرحلة 5 — لوحة الإدارة المتقدمة (Phase 5)

## ⚙️ منطق العمل (Business Logic Flow)

يسير العميل في الرحلة التالية داخل النظام:
1.  **التسجيل:** عبر Adtopia webhook أو تسجيل مباشر.
2.  **التوثيق:** إدخال البيانات القانونية ورفع الوثائق (أو طلب استخراج).
3.  **المدخلات التجارية:** وصف النشاط والجمهور المستهدف.
4.  **التوليد الآلي:** Gemini AI يقترح أسماء + ألوان + شعار.
5.  **المراجعة:** العميل يراجع ويعدل الهوية المقترحة قبل الاعتماد.
6.  **الاعتماد:** يتم إنشاء تذكرة رسمية ونقلها لمدير الحسابات.
7.  **لوحة التتبع:** العميل يتابع حالة طلبه + حالة الوثائق (قيد المراجعة / تمت المراجعة).
8.  **الإدارة:** الموظفون يديرون التذاكر عبر Dashboard مع أرشفة وحذف وتعيين.
